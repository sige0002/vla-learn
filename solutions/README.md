# 解答の使い方（solutions）

ここには [`../exercises/`](../exercises/) の模擬問題に対する解答（正解コードと解説）が入っています。
**先に自分で解いてから**、答え合わせに使ってください。

## まず自分で解く → それから比較

解答は「写すもの」ではなく「自分の答えと突き合わせるもの」です。次の順で使うと効果的です。

1. 対応する演習 [`../exercises/mX/`](../exercises/) を、まず自分で解く。
2. 詰まっても 15〜20 分は粘る。特に shape 確認・穴埋め・バグ修正は「自分で形を追う」ことが目的です。
3. 解けたら、または十分粘ったら、ここの `mX/` を開いて比較する。
4. コードが合っていても**解説を読む**。各解答には「**なぜその shape になるか / なぜ loss が下がるべきか**」の
   短い説明が付いています。コードの正誤だけでなく、理由まで一致しているかを確認してください。

> 答えを先に見ると、型 1〜3（shape・穴埋め・バグ修正）の練習効果がほとんど失われます。
> 自由実験（型 5）は「正解」が一つではないので、解答は考え方の一例として読んでください。
